Explication des pages principales de l'application Spotlight

1. Page de Login (app/(auth)/login.tsx)

Cette page permet à l'utilisateur de se connecter via Google grâce à Clerk. Elle comprend :

  • Un logo et le nom de l'application
  • Un slogan "don't miss anything"
  • Une illustration centrale
  • Un bouton Continue with Google qui déclenche l'authentification OAuth
  • Un texte d'information sur les conditions d'utilisation
{/*
- Utilise useOAuth de Clerk pour la connexion Google
- Redirige l'utilisateur vers /(tabs) après connexion
- Utilise des styles personnalisés (auth.style.ts)
*/}

2. Layout d'authentification (app/(auth)/_layout.tsx)

Ce layout gère l'affichage des pages d'authentification et le chargement de la police personnalisée. Il masque le header et applique un fond noir. 

{/*
- Charge la police JetBrainsMono-Medium
- Utilise Stack de expo-router pour la navigation
- Masque le header
*/}

3. Layout principal et protection des routes (app/_layout.tsx & components/InitialLauyot.tsx)

Le layout principal englobe toute l'application et applique la logique de protection des routes :

  • Si l'utilisateur n'est pas connecté, il est redirigé vers /(auth)/login
  • Si l'utilisateur est connecté, il est redirigé vers /(tabs)
  • Utilise ClerkProvider pour fournir le contexte d'authentification
{/*
- useAuth pour vérifier l'état de connexion
- useSegments et useRouter pour la navigation conditionnelle
- Stack pour la navigation principale
*/}

4. Page de Tabs (app/(tabs)/_layout.tsx)

Cette page définit la navigation par onglets (Tabs) en bas de l'application. Chaque onglet correspond à une section (Accueil, Bookmarks, Création, Notifications, Profil).

  • Personnalisation des icônes et couleurs
  • Masquage du label des onglets
  • Style sombre et moderne
{/*
- Utilise Tabs et Tabs.Screen de expo-router
- Icônes Ionicons pour chaque onglet
- Style défini dans tabBarStyle
*/}

5. Styles personnalisés (styles/auth.style.ts)

Ce fichier centralise tous les styles utilisés pour la page de login et d'autres composants d'authentification.

  • Couleurs, polices, marges, paddings, ombres
  • Styles pour le bouton Google, l'illustration, le texte, etc.
Remarque : L'application utilise Clerk pour l'authentification sécurisée, expo-router pour la navigation, et applique une séparation claire entre les pages publiques (auth) et privées (tabs).
